MySQL索引

  1. 2021-11-09
  2. 414字
  3. 2 min read

一：什么是索引

  1. 索引是帮助MySQL高效获取数据的数据结构。
  2. 索引往往存储在磁盘上的文件中
  3. 索引中包括:聚集索引，覆盖索引，组合索引，前缀索引，唯一索引等，默认都是使用B+树结构组织索引

二：索引的弊端

  1. 占磁盘空间。
  2. 索引不是所有情况都使用，比如：少量数据、频繁变化的字段、很少使用的字段
  3. 索引会降低增删改的效率

二、MySQL索引类型

  1. FULLTEXT: 即为全文索引，目前只有MyISAM引擎支持。其可以在CREATE TABLE ，ALTER TABLE ，CREATE INDEX 使用，不过目前只有 CHAR、VARCHAR ，TEXT 列上可以创建全文索引。全文索引为了解决WHERE name LIKE “%word%"这类针对文本的模糊查询效率较低的问题。
  2. HASH：由于HASH的唯一(几乎100%的唯一)及类似键值对的形式，很适合作为索引。HASH索引可以一次定位，不需要像树形索引那样逐层查找,因此具有极高的效率。但是，这种高效是有条件的，即只在“=”和“in”条件下高效，对于范围查询、排序及组合索引仍然效率不高。
  3. BTREE：BTREE索引就是一种将索引值按一定的算法，存入一个树形的数据结构中(二叉树)，每次查询都是从树的入口root开始，依次遍历node，获取leaf。这是MySQL里默认和最常用的索引类型。
  4. RTREE：RTREE在MySQL很少使用，仅支持geometry数据类型，相对于BTREE，RTREE的优势在于范围查找。
